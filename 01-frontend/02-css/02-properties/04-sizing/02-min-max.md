# CSS `min-*` / `max-*` (min-width, max-width, min-height, max-height)

## Визначення

Властивості **`min-*` і `max-*`** задають **обмеження розмірів елемента**:

* `min-width` → мінімальна ширина елемента
* `max-width` → максимальна ширина елемента
* `min-height` → мінімальна висота елемента
* `max-height` → максимальна висота елемента

Ці властивості не задають остаточний розмір елемента напряму. Вони тільки обмежують діапазон, в якому браузер може обчислити фактичну ширину або висоту.

---

## Базовий синтаксис

```css
.box {
    width: 100%;
    max-width: 1200px;
}
```

---

## Як працює (ключова ідея)

```text
min ≤ actual size ≤ max
```

Браузер обирає розмір елемента в межах допустимого діапазону.

---

## min-width

```css
.box {
    min-width: 200px;
}
```

`min-width` задає найменшу можливу ширину. Якщо внутрішній вміст або властивість `width` менші, елемент все одно не стане ширшим за цей мінімум.

---

### Приклад

```css
.box {
    width: 100px;
    min-width: 200px;
}
```

результат: **200px**

`width` намагається встановити 100px, але `min-width` підвищує цей розмір до 200px.

---

## max-width

```css
.box {
    max-width: 500px;
}
```

`max-width` задає найбільшу дозволену ширину. Якщо елемент намагається розтягнутися більше, браузер обмежує його до цього значення.

---

### Приклад

```css
.box {
    width: 1000px;
    max-width: 500px;
}
```

результат: **500px**

`width` встановлює 1000px, але `max-width` зменшує фактичну ширину до 500px.

---

## min-height / max-height

```css
.box {
    min-height: 100px;
    max-height: 300px;
}
```

`min-height` задає мінімальну висоту, щоб елемент не став занадто низьким.
`max-height` задає максимальну висоту, щоб елемент не став занадто високим.

Ці властивості працюють схоже на `min-width` / `max-width`, але для висоти.

---

## Пріоритет (дуже важливо)

```text
min-width > width > max-width
```

Якщо значення `width` виходить за межі заданих `min-width` або `max-width`, браузер обере значення з урахуванням обмежень.

---

### Приклад

```css
.box {
    width: 100px;
    min-width: 200px;
    max-width: 150px;
}
```

результат: **200px**

`min-width` переважає, бо мінімальне обмеження сильніше за `width` і `max-width`.

---

## Одиниці

```css
min-width: 200px;
max-width: 80%;
max-height: 100vh;
```

Можна використовувати різні одиниці: пікселі, відсотки, `vw`, `vh`, `em`, `rem` та інші.

---

## Найважливіший кейс: container

---

### Погано

```css
.container {
    width: 1200px;
}
```

Таке рішення не адаптується до маленьких екранів.

---

### Добре

```css
.container {
    max-width: 1200px;
    margin: 0 auto;
}
```

Контейнер займає доступний простір, але не ширше за 1200px.

---

## Зображення

```css
img {
    max-width: 100%;
    height: auto;
}
```

Зображення не виходить за межі контейнера і зберігає пропорції.

---

## overflow + max-height

```css
.box {
    max-height: 200px;
    overflow: auto;
}
```

Якщо вміст перевищує висоту, зʼявляється прокрутка.

---

## Flexbox нюанси (дуже важливо)

---

### min-width за замовчуванням

```css
.item {
    min-width: auto;
}
```

За замовчуванням flex-елементи можуть мати мінімальну ширину, що залежить від їх вмісту. Це може призводити до “ламання” макета при стисненні.

---

### Фікс

```css
.item {
    min-width: 0;
}
```

Це дозволяє flex-елементу стискатися і не виходити за межі контейнера.

---

## Grid нюанси

```css
grid-template-columns: minmax(200px, 1fr);
```

`minmax()` встановлює мінімальне і максимальне значення для розміру колонки в Grid.

---

## minmax() (важливо)

```css
grid-template-columns: minmax(200px, 500px);
```

Колонка буде не менше 200px і не більше 500px.

---

## Практичні приклади

---

### Responsive container

```css
.container {
    max-width: 1200px;
    margin: 0 auto;
}
```

---

### Card

```css
.card {
    min-height: 200px;
}
```

---

### Scroll block

```css
.list {
    max-height: 300px;
    overflow: auto;
}
```

---

### Adaptive grid

```css
.container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
}
```

## Часті питання на співбесіді

* min vs max?
* яка властивість контролює мінімальний розмір?
* яка властивість контролює максимальний розмір?
* чому `min-width: 0` важливий у flex?
* як працює `minmax()`?

---

## Часті помилки

* використовують `width` замість `max-width`
* забувають про `min-width` у flex
* не розуміють пріоритет обмежень
* не враховують `overflow` з `max-height`

---

## Best Practices

* використовувати `max-width` для контейнерів
* використовувати `min-height` для стабільності UI
* використовувати `min-width: 0` у flex
* комбінувати з grid (`minmax`)

---

## Висновок

`min-*` / `max-*`:

* контролюють адаптивність
* важливі для responsive layout
* критичні у flex/grid
