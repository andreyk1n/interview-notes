# CSS Gradients

## Визначення

**Gradients (градієнти)** — це CSS-функції, які дозволяють створювати плавний перехід між кольорами без використання зображень.

Вони використовуються як значення для:

```css
background
background-image
```

---

## Типи градієнтів

---

## 1. Linear Gradient

```css
background: linear-gradient(red, blue);
```

Пояснення: `linear-gradient()` створює лінійний перехід від одного кольору до іншого вздовж прямої. Перший аргумент — початковий колір, останній — кінцевий. Можна додавати кілька кольорів для складніших переходів.

---

### Напрямок

```css
background: linear-gradient(to right, red, blue);
```

Пояснення: `to right` задає напрямок переходу зліва направо. За замовчуванням напрямок — зверху вниз. Властивість всередині функції визначає, куди прямує градієнт.

---

| Значення  | Опис                    |
| --------- | ----------------------- |
| to right  | перехід йде зліва направо |
| to left   | перехід йде справа наліво |
| to bottom | перехід йде зверху вниз (значення за замовчуванням) |
| to top    | перехід йде знизу вгору |

---

### Кути

```css
background: linear-gradient(45deg, red, blue);
```

Пояснення: `45deg` задає кут нахилу градієнта від горизонталі. Точка 0deg відповідає напрямку вгору. Значення задає кут, під яким відбувається перехід.

---

## Color Stops

```css
background: linear-gradient(red 0%, blue 100%);
```

Пояснення: `color stop` вказує позицію кольору у градієнті. Значення від 0% до 100% визначає точку початку або кінця конкретного кольору. Це дозволяє точно контролювати, де починається і закінчується кожний колір.

---

### Приклад

```css
background: linear-gradient(
    red 0%,
    yellow 50%,
    blue 100%
);
```

Пояснення: три стопи задають червоний колір на початку, жовтий посередині і синій в кінці. Між стопами відбувається плавне змішування кольорів.

---

### Жорсткі переходи

```css
background: linear-gradient(
    red 50%,
    blue 50%
);
```

Пояснення: коли два кольори мають однакову позицію, перехід стає різким без плавного змішування. Це корисно для створення чітких сегментів.

---

## 2. Radial Gradient

```css
background: radial-gradient(red, blue);
```

Пояснення: `radial-gradient()` створює градієнт, що розповсюджується від центральної точки назовні. Перший колір знаходиться в центрі, останній — на краю.

---

### Форма

```css
radial-gradient(circle, red, blue);
radial-gradient(ellipse, red, blue);
```

Пояснення: `circle` задає кругову форму, `ellipse` — еліптичну форму. Форма визначає контур, в якому відбувається перехід.

---

### Позиція

```css
radial-gradient(circle at center, red, blue);
radial-gradient(circle at top right, red, blue);
```

Пояснення: `at center` або `at top right` визначає місце старту градієнта. Це дозволяє розташувати центр радіального градієнта в будь-якій частині елемента.

---

## 3. Conic Gradient

```css
background: conic-gradient(red, blue);
```

Пояснення: `conic-gradient()` створює кутовий градієнт навколо центральної точки, де кольори змінюються за обертанням. Виглядає як сектори кола.

---

### Початок

```css
conic-gradient(from 90deg, red, blue);
```

Пояснення: `from 90deg` задає напрямок, з якого починається перший колір. Це дозволяє повертати весь градієнт.

---

### Сегменти

```css
conic-gradient(
    red 0deg 120deg,
    blue 120deg 240deg,
    green 240deg 360deg
);
```

Пояснення: кожному кольору можна задати початковий і кінцевий кут. Це створює чіткі кольорові сегменти.

---

## Transparency

```css
background: linear-gradient(
    rgba(0,0,0,0.5),
    rgba(0,0,0,0)
);
```

Пояснення: прозорі кольори (`rgba` або `hsla`) дозволяють створювати напівпрозорі градієнти. Це корисно для накладання на інші фони та зображення.

---

## Multiple gradients

```css
background:
    linear-gradient(red, blue),
    radial-gradient(circle, yellow, green);
```

Пояснення: у `background` можна використовувати кілька градієнтів. Перший в списку буде зверху, наступні — нижче. Це дає змогу комбінувати різні ефекти.

---

## Repeating gradients

---

### repeating-linear-gradient

```css
background: repeating-linear-gradient(
    45deg,
    red 0 10px,
    blue 10px 20px
);
```

Пояснення: `repeating-linear-gradient()` створює повторюваний лінійний візерунок. Кольори повторюються через заданий інтервал.

---

### repeating-radial-gradient

```css
background: repeating-radial-gradient(
    circle,
    red 0 10px,
    blue 10px 20px
);
```

Пояснення: `repeating-radial-gradient()` створює повторюваний радіальний візерунок. Кольори повторюються у концентричних колах.

---

## Практичні приклади

---

## 1. Overlay

```css
background:
    linear-gradient(rgba(0,0,0,0.5), rgba(0,0,0,0.5)),
    url("bg.jpg");
```

Пояснення: напівпрозорий градієнт накладається на зображення. Це зменшує яскравість фону і підвищує контраст тексту.

---

## 2. Кнопка

```css
button {
    background: linear-gradient(to right, #ff7e5f, #feb47b);
}
```

Пояснення: градієнт по горизонталі робить кнопку більш виразною. `to right` задає напрямок переходу зліва направо.

---

## 3. Stripes

```css
background: repeating-linear-gradient(
    90deg,
    black 0 10px,
    white 10px 20px
);
```

Пояснення: створює горизонтальні смуги шириною 10px, які повторюються. Це корисно для фонів та декоративних елементів.

---

## 4. Loader (conic)

```css
background: conic-gradient(
    from 0deg,
    transparent,
    blue
);
```

Пояснення: конічний градієнт може імітувати індикатор або loader, де один сектор пофарбований, а інший прозорий.

---

## 5. Градієнтний текст

```css
.text {
    background: linear-gradient(red, blue);
    -webkit-background-clip: text;
    color: transparent;
}
```

Пояснення: фон-градієнт застосовується до тексту за допомогою `background-clip: text`. Колір тексту робиться прозорим, щоб видно було тільки градієнт.

---

## Performance

- градієнти змінюються без додаткових HTTP-запитів
- не потребують завантаження зображень
- браузери часто рендерять їх ефективніше з GPU

---

## Часті питання на співбесіді

- в чому різниця між `linear-gradient()` та `radial-gradient()`?
- що таке `color stop` і як його використовувати?
- як зробити смуги за допомогою градієнта?
- як створити накладення (`overlay`)?
- як працює `conic-gradient()`?

---

## Часті помилки

- не задають напрямок у лінійному градієнті
- не контролюють позиції стопів
- забувають про прозорість
- не розуміють порядок накладення при multiple backgrounds

---

## Best Practices

- використовувати градієнти замість зображень, коли це можливо
- комбінувати градієнти з іншими властивостями `background`
- використовувати напівпрозорі кольори для overlay
- контролювати `color stop` для точного результату

---

## Висновок

Gradients:

- потужний інструмент для UI
- можуть замінювати зображення
- покращують продуктивність
