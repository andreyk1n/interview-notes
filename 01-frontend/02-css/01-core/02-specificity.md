# Специфічність CSS

## Визначення

**Специфічність** — це правило, за яким браузер визначає, **який CSS-стиль застосувати**, якщо кілька селекторів впливають на один і той самий елемент.

Іншими словами:
**хто "сильніший" у конфлікті стилів**

---

## Як працює

Якщо є кілька правил:

```css
p { color: blue; }
.text { color: red; }
#title { color: green; }
```

```html
<p id="title" class="text">Hello</p>
```

Результат:

```id="res1"
color: green;
```

Бо `id` має більшу специфічність

---

## Формула специфічності

Специфічність записується як:

```id="formula"
(inline, id, class, element)
```

---

## Вага селекторів

| Тип                              | Вага |
| -------------------------------- | ---- |
| Inline styles                    | 1000 |
| ID                               | 100  |
| Class / attribute / pseudo-class | 10   |
| Element / pseudo-element         | 1    |

---

## Приклади

---

### Простий

```css
p {}
```

(0,0,0,1)

---

### Клас

```css
.text {}
```

(0,0,1,0)

---

### ID

```css
#app {}
```

(0,1,0,0)

---

### Комбінований

```css
#app .text p {}
```

(0,1,1,1)

---

## Порівняння

```css
#id {}
.class {}
```

Перемагає `#id`

---

```css
.class1.class2 {}
.class1 {}
```

Перемагає `.class1.class2`

---

## Inline styles

```html
<p style="color: red;"></p>
```

(1,0,0,0)

Найвища специфічність (крім !important)

---

## !important

```css
p {
    color: red !important;
}
```

Перекриває specificity

Але:

* ускладнює підтримку
* не best practice

---

## Cascade (каскад)

Якщо specificity однакова:

перемагає **останнє правило**

```css
p { color: blue; }
p { color: red; }
```

red

---

## Псевдокласи та псевдоелементи

```css
a:hover {}
```

псевдоклас = 10

---

```css
p::before {}
```

псевдоелемент = 1

---

## :is(), :where(), :not()

---

### :is()

```css
:is(.a, .b) {}
```

бере specificity найсильнішого

---

### :where()

```css
:where(.a) {}
```

specificity = 0

Дуже корисно

---

### :not()

```css
:not(.a) {}
```

не додає вагу, але враховує внутрішній селектор

---

## Важливі нюанси

---

### Атрибути

```css
input[type="text"] {}
```

= class (10)

---

### Комбінатори не впливають

```css
div > p {}
```

не додають вагу

---

## Практичні кейси

---

### Погано

```css
#app .header .nav ul li a {}
```

надто велика специфічність

---

### Добре

```css
.nav__link {}
```

BEM

---

## Як керувати специфічністю

* використовувати класи
* уникати ID
* не вкладати глибоко
* використовувати :where()
* уникати !important

---

## Часті питання на співбесіді

* що таке specificity?
* як вона рахується?
* що сильніше: class чи id?
* як працює !important?
* що перемагає при однаковій специфічності?

---

## Часті помилки

* зловживання !important
* використання ID
* складні селектори
* нерозуміння cascade

---

## Best Practices

* використовувати класи (BEM)
* тримати низьку специфічність
* не використовувати !important без потреби
* писати прості селектори
* контролювати cascade

---

## Висновок

Специфічність:

* визначає, який стиль застосовується
* працює разом з cascade
* критично важлива для масштабованого CSS
